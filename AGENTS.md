# AGENTS.md — AI Coding Agent Guide

## Project Overview
**STM/TMS** — a Yii2 PHP business application (PHP ≥ 8.1) for managing quotations, purchase orders, inventory, delivery receipts, proforma invoices, and asset cards. Domain language mixes Bahasa Indonesia and English (e.g., `nomor`, `barang`, `gudang`, `rekening`).

## Architecture

### Model Layering (critical — do not conflate)
| Directory | Purpose | Edit? |
|---|---|---|
| `models/base/` | Auto-generated by **giiant** (Gii). Never edit manually. | ❌ |
| `models/` | Custom extensions of base models; add business logic, traits, constants here. | ✅ |
| `models/active_queries/` | Custom query classes returned by `Model::find()`; put reusable scopes/maps here. | ✅ |
| `models/search/` | Search models for `GridView` data providers. | ✅ |
| `models/form/` | Form-only models (no DB table). | ✅ |

Regenerating a base model overwrites `models/base/ModelName.php`; changes belong in `models/ModelName.php`.

### Master-Detail Pattern
Complex documents (Quotation, PurchaseOrder, etc.) use a master-detail architecture:
- Models implement `MasterDetailsInterface` (`createWithDetails`, `updateWithDetails`).
- Detail sections are split into **Component classes** (`components/BarangQuotation.php`, `components/ServiceQuotation.php`, etc.) that implement `CreateModelDetails` / `UpdateModelDetails` / `DeleteModelDetails` interfaces.
- `models/Tabular.php` → `Tabular::createMultiple()` handles dynamic form rows (used by dynamic form widget `ahmadfadlydziljalal/yii2-dynamicform-v2`).
- Multi-scenario models use `SCENARIO_*` class constants (e.g., `Quotation::SCENARIO_CREATE_BARANG_QUOTATION`).
- Multi-step stock workflows use the same scenario-driven approach, but with form models in `models/form/` plus orchestration components such as `components/WizardStockPerGudangBarangMasukDariTandaTerimaPoForm.php`; see `controllers/StockPerGudangController.php` for the step-1 / step-2 controller pattern.
- Several index/view flows use AJAX Bootstrap modals instead of full-page CRUD: controllers return JSON with `renderAjax(...)`, views include `app\widgets\modal_ajax\ModalAjaxWidget`, and shared action-button defaults (`role => 'modal-remote'`) live in `config/container.php`.

### Traits for Shared Behaviour
Models use PHP traits to separate concerns:
- `QuotationBarangTrait`, `QuotationServiceTrait`, `QuotationDeliveryReceiptTrait` — line-item logic
- `NomorSuratTrait` — formats `nomor` (document number) for display
- `ProformaDebitNoteDetailBarangTrait`, etc.

### Enums (PHP 8.1)
Located in `enums/`. Prefer them over raw strings:
- `SessionSetFlashEnum` — flash keys: `success`, `info`, `danger`, `warning`
- `TipePergerakanBarangEnum`, `TandaTerimaStatusEnum`, `DeliveryReceiptEnum`, etc.

## Configuration
All web config is assembled in `config/web.php` from split files in `config/`. Key components:
- **DB**: the main connection uses `.env` values from `getenv('DB_DSN')`, `getenv('DB_USER')`, `getenv('DB_PASSWORD')`; a secondary `supportDb` connection uses `getenv('DB_DSN_SUPPORT')` and backs session/cache/log storage (`config/session.php`, `config/cache.php`, `config/log.php`). Schema cache is enabled only in non-dev.
- **Auth/RBAC**: `yii\rbac\DbManager`, managed via `mdmsoft/yii2-admin` (`admin` module). Default role: `user-default`.
- **External auth**: login is delegated to the SIHRD OAuth2 client in `config/auth_client_collection.php` / `components/SihrdAuthClient.php`; the flow is wired through `SiteController::actionLogin()` and `LoginForm::loginByOauth2ResourceOwnerPassword()`. Relevant env vars include `HRD_CLIENT_ID`, `HRD_CLIENT_SECRET`, `HRD_AUTH_URL`, `HRD_TOKEN_URL`, `HRD_API_BASE_URL`, and `HRD_API_USER_INFO`.
- **Access control**: `mdm\admin\components\AccessControl` applied globally; `site/*` and `dark-light-toggle/*` are open to guests.
- **File storage**: DigitalOcean Spaces (S3-compatible) via `aws` component (`components/AwsS3Filesystem.php`). Bucket: `files.tsurumaru.online`, region `sgp1`.
- **PDF**: two components — `pdf` (plain) and `pdfWithLetterhead` using `kartik-v/yii2-mpdf`.
- **Container defaults**: `config/container.php` centralizes shared UI defaults for `GridView`, `kartik\grid\GridView`, `ActionColumn`, `DetailView`, `DatePicker`, `DateTimePicker`, and `ExportMenu`; prefer changing shared presentation there before overriding individual views.
- **Bootstrap hooks**: `config/bootstrap.php` boots `app\components\bootstraps\PdfProperties`, which injects shared PDF header/footer settings.
- **Timezone**: `Asia/Jakarta`. Date display format: `d-m-Y` (day-month-year).

## Developer Workflows

### Local Development (Docker)
```bash
# Start all services (MySQL, PHP/Apache, Gulp/Node, Chrome, Adminer)
docker-compose up -d

# Run migrations inside PHP container
docker-compose exec php php yii migrate

# Open shell in PHP container
docker-compose exec php bash
```
Services: MySQL 8 (`DB_PORT`), Apache (`WEB_SERVER_PORT`), Adminer (`ADMINER_PORT`), BrowserSync (3000/3001).

### Frontend (Gulp + Sass)
Source: `themes/v2/assets/scss/` and `themes/v2/assets/js/`
Built to: `themes/v2/dist/css/` and `themes/v2/dist/js/`
```bash
# Inside gulp container or locally (node 16)
gulp          # build + watch
gulp styles   # compile Sass only
gulp scripts  # minify JS only
```
`assets/ThemeAsset.php` serves files from `themes/v2/dist/`, so edits under `themes/v2/assets/` need a rebuild before they appear in the app. The default Gulp task also watches `views/**/*.php` and proxies the PHP container at `http://php:80` for BrowserSync.

### Code Generation (Gii)
Custom generator `dzilcrud` at `generators/dzilcrud/` with two templates:
- `master-details` — generates master + one detail layer
- `master-details-details` — generates master + two detail layers

Access Gii at `/gii` in dev mode. Base models land in `models/base/`; always extend in `models/`.

### Running Tests
```bash
# Inside PHP container
php vendor/bin/codecept run        # all suites
php vendor/bin/codecept run functional  # functional only
php vendor/bin/codecept run unit   # unit only
php vendor/bin/codecept run acceptance  # requires Chrome container
```
Test config: `config/test.php`. Selenium endpoint: `http://chrome:4444`.
If you add or remove Codeception modules, regenerate actor classes with `php vendor/bin/codecept build`.

### Migrations
```bash
php yii migrate              # apply all pending
php yii migrate/create Name  # create new migration
```
Naming convention: `m{YYMMDD}_{HHMMSS}_{PascalCaseDescription}.php`

## Key Files & Directories
| Path | Role |
|---|---|
| `config/web.php` | Main web app config (assembles all `config/*.php`) |
| `config/modules.php` | Registered modules (admin, datecontrol, settings, gridview, gridviewPrint) |
| `config/as_access.php` | Global RBAC access filter config |
| `config/auth_client_collection.php` | External SIHRD OAuth2 client wiring |
| `config/bootstrap.php` | App bootstrap classes (including shared PDF properties) |
| `config/container.php` | Global DI definitions for grid/form/widget defaults |
| `models/MasterDetailsInterface.php` | Contract for master-detail models |
| `models/Tabular.php` | Dynamic multi-row form helper |
| `models/active_queries/` | Reusable query scopes and `map()` helpers returned by `Model::find()` |
| `components/` | Business logic components (per-document detail managers) |
| `enums/` | PHP 8.1 backed enums for domain constants |
| `themes/v2/` | Active UI theme (Sass + JS source + dist) |
| `widgets/modal_ajax/` | Shared AJAX modal widget used by modal-based CRUD/index flows |
| `generators/dzilcrud/` | Custom Gii CRUD generator templates |
| `migrations/` | All DB migrations in chronological order |
